# TypeScript Decorators

TypeScript decorators are a meta-programming feature that enables annotation and modification of classes and class members. They use the `@expression` syntax and require the `experimentalDecorators` compiler option.

```json
{
  "compilerOptions": {
    "experimentalDecorators": true
  }
}
```

Note: This covers the experimental Stage 2 decorator spec, which is what Angular uses. Stage 3 decorators became available in TypeScript 5.0 and have a different API.

## Decorator Types

### Class Decorators

Applied to class constructors. Can observe, modify, or replace class definitions. The class decorator is called with the constructor of the decorated class.

```ts
function sealed(constructor: Function) {
  Object.seal(constructor);
  Object.seal(constructor.prototype);
}

@sealed
class BugReport {
  type = 'report';
  title: string;
  constructor(t: string) { this.title = t; }
}
```

If the class decorator returns a value, it replaces the class declaration with the returned constructor function.

### Method Decorators

Applied to method property descriptors. Receives three arguments: the target (prototype for instance methods, constructor for static methods), the property name, and the property descriptor.

```ts
function enumerable(value: boolean) {
  return function (target: any, propertyKey: string, descriptor: PropertyDescriptor) {
    descriptor.enumerable = value;
  };
}

class Greeter {
  @enumerable(false)
  greet() { return 'Hello!'; }
}
```

### Accessor Decorators

Applied to getter/setter property descriptors. Cannot be applied to both the get and set accessor of a single member simultaneously — the decorator must be applied to the first accessor specified.

### Property Decorators

Applied at property declarations. Receives two arguments: the target and the property name. No property descriptor is available.

```ts
function format(formatString: string) {
  return function (target: any, propertyKey: string) {
    // record that this property should be formatted
  };
}

class Greeter {
  @format('Hello, %s')
  greeting: string;
}
```

### Parameter Decorators

Applied to function parameters. Receives three arguments: the target, the method name, and the ordinal index of the parameter.

```ts
function required(target: Object, propertyKey: string | symbol, parameterIndex: number) {
  // record that this parameter is required
}

class Greeter {
  greet(@required name: string) { return 'Hello ' + name; }
}
```

## Decorator Factories

A decorator factory is a function that returns the actual decorator, enabling parameterisation:

```ts
function color(value: string) {
  return function (target: any) {
    // apply decorator using `value`
  };
}

@color('red')
class MyClass {}
```

## Evaluation Order

When multiple decorators apply to a single declaration:
- Expressions evaluate **top-to-bottom**
- Functions execute **bottom-to-top** (like mathematical function composition)

The evaluation sequence within a class:
1. Instance member decorators (parameters, methods, accessors, properties)
2. Static member decorators
3. Constructor parameter decorators
4. Class decorators

## Metadata

The `reflect-metadata` library enables runtime type information through the `emitDecoratorMetadata` compiler option, allowing decorators to access design-time type data at runtime.

```json
{
  "compilerOptions": {
    "experimentalDecorators": true,
    "emitDecoratorMetadata": true
  }
}
```
